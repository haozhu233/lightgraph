# Vignette embed generators

The live figures in `docs/source/vignette_python.rst` and
`docs/source/vignette_r.rst` are generated by the real Python and R
bindings from the OpenFlights dataset, then slimmed down to share the JS
assets in `docs/source/_static/demo/` (and, for full-network figures, a
single fetched edge list under `_static/vignettes/data/`).

To regenerate after changing a binding or the vignette code:

```bash
# 1. Fetch the dataset (any scratch dir)
curl -sL https://raw.githubusercontent.com/jpatokal/openflights/master/data/airports.dat -o /tmp/lg/airports.dat
curl -sL https://raw.githubusercontent.com/jpatokal/openflights/master/data/routes.dat   -o /tmp/lg/routes.dat

# 2. Python embeds (py_*.html + data/flights_edges_py.json)
PYTHONPATH=python uv run --project python --with pandas \
    python docs/tools/gen_vignette_embeds.py docs/source/_static/vignettes /tmp/lg

# 3. R embeds (r_*.html + data/flights_edges_r.json); expects an R library
#    containing the lightgraph package under <scratch>/rlib, e.g.
#    R CMD INSTALL -l /tmp/lg/rlib R
Rscript docs/tools/gen_vignette_embeds.R /tmp/lg docs/source/_static/vignettes
```

Keep the code inside the generators in sync with the code shown in the
vignette pages — the pages claim the figures are the output of exactly
that code.

## Example / custom-JS embeds

The figures on `docs/source/examples.rst` and
`docs/source/custom_js.rst` come from the bundled datasets (no download
needed) and are generated by:

```bash
PYTHONPATH=python uv run --project python \
    python docs/tools/gen_example_embeds.py docs/source/_static/examples
```

The same sync rule applies: the `net_vis()` calls in the generator must
match the code blocks shown on those pages.
